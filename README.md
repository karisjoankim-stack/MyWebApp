# 🐍 Python Sandbox & Runner Web App

A modern, full-stack web application that allows users to write, run, and analyze Python code directly in the browser. It features a Monaco-based code editor, real-time code execution with output capture, package availability checking, and dynamic rendering of generated media (such as charts or images generated by `matplotlib` or `pillow`).

The application is fully containerized using **Docker** and **Docker Compose** for seamless setup and isolated execution environments.

---

## 🚀 Key Features

* **Monaco Editor Integration**: Features a rich, IDE-like code editing experience directly in the browser, complete with syntax highlighting.
* **Sandboxed Python Execution**: Safely run Python scripts in a dedicated workspace directory on the backend, capturing stdout, stderr, execution times, and exit codes.
* **Automatic Media Extraction**: Automatically scans the execution environment for generated media files (e.g., `.png`, `.jpg`, `.mp4` generated by data science scripts) and displays them to the user.
* **Package Check Dashboard**: Instantly inspects the server environment to verify if common data science libraries (`numpy`, `pandas`, `matplotlib`, `pillow`, `scipy`, `scikit-learn`, `requests`) are installed.
* **Run History**: View, re-run, and browse through past executions and their source code.
* **Dockerized Setup**: Multi-container orchestrations for running both backend API and React frontend simultaneously with live hot-reloading.

---

## 🛠️ Tech Stack

### Frontend
* **Core**: React 19 & React Router 7
* **Build System**: Vite
* **Code Editor**: `@monaco-editor/react`
* **Icons**: `lucide-react`
* **HTTP Client**: `axios`

### Backend
* **Runtime**: Node.js & Express
* **Execution System**: Sandboxed process spawning (`child_process`) running `python3`
* **Storage**: Temp directory manager with auto-pruning to keep disk space usage low.

## 🏃 Getting Started

You can run this application either using **Docker Compose** (recommended) or **locally / natively**.

### Option A: Running with Docker Compose (Recommended)
This starts the entire application stack in containers with hot-reloading enabled. Make sure you have Docker installed.

1. **Start the containers**:
   ```bash
   docker-compose up --build
   ```
2. **Access the application**:
   * **Frontend**: [http://localhost:5173](http://localhost:5173)
   * **Backend API**: [http://localhost:3001](http://localhost:3001)

---

### Option B: Running Natively (Without Docker)
Make sure you have Node.js (v18+) and Python 3 installed on your system.

#### 1. Setup the Backend
1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the backend server:
   ```bash
   node index.js
   ```
   *(Running on [http://localhost:3001](http://localhost:3001))*

#### 2. Setup the Frontend
1. Open a new terminal and navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the dev server:
   ```bash
   npm run dev
   ```
   *(Running on [http://localhost:5173](http://localhost:5173))*

## 🔌 API Documentation (Backend)

The backend server exposes the following HTTP endpoints:

| Endpoint | Method | Description |
| :--- | :--- | :--- |
| `/api/run` | `POST` | Executes Python code sent in request body (`{ "code": "..." }`). Returns execution stats, outputs, and any generated media URLs. |
| `/api/packages` | `GET` | Checks and returns the installation status of standard data science packages (`numpy`, `pandas`, etc.) in the backend environment. |
| `/api/history` | `GET` | Fetches a historical list of all recent executions, their source code, run times, and outcomes. |
| `/api/recent-code` | `GET` | Returns the source code of the most recently executed run. |
| `/api/results/:runId/:filename` | `GET` | Serves statically generated output files (like charts or images) for a specific run. |

## 🔒 Security Note
*This runner is meant for local experimentation, research, and developer environments. Since it executes arbitrary Python scripts on the host system (or inside a docker container), it should not be exposed directly to the open internet without adding secure container sandboxing/gvisor layers.*
